## 📚 Topics Covered

### Module 1 — Introduction to Data Mining

**Why Data Mining?**
- The explosion of digital data has created "data tombs" — massive databases rarely revisited for insight
- Data mining tools extract hidden patterns and turn raw data into actionable knowledge ("golden nuggets")
- Applications span business strategy, medical research, engineering, and market analysis

**Key Concepts:**
- Evolution of database technology → data warehousing → data mining
- Difference between **KDD (Knowledge Discovery in Databases)** and **Data Mining**
  - KDD = the full process of extracting knowledge
  - Data Mining = applying specific mining algorithms within that process
- General architecture of a data mining system
- Types of data that can be mined (relational, transactional, spatial, time-series, etc.)
- Potential applications and known limitations of data mining

---

### Module 2 — Data Preparation and Preprocessing

**Why Preprocess?**
Raw data is almost never analysis-ready — it contains noise, missing values, inconsistencies, and redundancies. Preprocessing makes data suitable for mining algorithms.

**Steps Covered:**

| Step | Description |
|------|-------------|
| **Data Cleaning** | Handle missing values, remove noise, resolve inconsistencies |
| **Data Integration** | Combine data from multiple sources into a unified view |
| **Data Transformation** | Normalization, aggregation, generalization |
| **Data Reduction** | Reduce volume while preserving analytical results (sampling, dimensionality reduction) |
| **Discretization** | Convert continuous values into intervals; concept hierarchy generation |

**Key Concepts:**
- Attribute types: qualitative (nominal, ordinal) vs quantitative (interval, ratio)
- Population vs sample — representative and unbiased sampling
- Measurement scales and attribute characteristics

## 💭 Reflection

Data Mining changed how I think about data — not as something to store, but as something to interrogate. Before this course, I understood databases and programming, but I had not thought systematically about how to extract meaning from large, messy datasets. The KDD framework gave me a structured way to approach that problem: you cannot jump straight to mining algorithms; you have to understand your data, clean it, and transform it first.

Module 2 on data preprocessing was the most practically useful part of the course for me. I had already encountered dirty data in other projects, but this was the first time I understood *why* each cleaning step matters — missing values bias results, inconsistent formats break pipelines, and unreduced data wastes computation. These lessons fed directly into how I approached the HPDP project (Carousell data cleaning) and the PPG industry pipeline.

The bigger takeaway was conceptual: data mining is not magic. It requires deliberate choices about what algorithm to apply, what the data actually represents, and what the results really mean. That critical mindset is something I carry into every data project now.
